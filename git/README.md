# Git Basics — Daily Workflow

## Overview

Конспект по основам Git и GitLab. Описан **идеальный процесс работы**, который покрывает ~80% задач разработчика/аналитика в команде.

---

## Standard Daily Workflow

### 1. Клонирование репозитория
```bash
git clone ssh_ссылка_на_репозиторий
cd название_репозитория
```

### 2. Создание и переход в новую ветку
```bash
git checkout -b название_ветки
```

### 3. Проверка изменений
```bash
git status
```
*Красные файлы = изменения не добавлены в коммит.*

### 4. Добавление изменений в коммит
```bash
git add файл          # конкретный файл
git add .             # все изменения в текущей папке
git add --all         # все изменения во всём репозитории
```

### 5. Коммит
```bash
git commit -m 'тип: краткое описание'
```

| Тип | Пример |
| :--- | :--- |
| **feat:** | `feat: add revenue calculation` |
| **fix:** | `fix: handle missing CustomerID` |
| **docs:** | `docs: update README with business insights` |
| **refactor:** | `refactor: rename column from total to sum` |

### 6. Отправка (пуш) в удалённый репозиторий
```bash
git push origin название_ветки
```
*После пуша появится ссылка для создания Merge Request (Pull Request).*

### 7. Создание Merge Request
1. Перейти по ссылке из терминала.
2. Выбрать: исходная ветка → целевая (`master`/`main`).
3. Добавить описание изменений.
4. Назначить ревьювера (если требуется).
5. Создать MR.

### 8. Обновление актуального состояния (перед началом работы)
```bash
git checkout master
git pull
git checkout свою_ветку
git merge master
```
*Зачем: чтобы ваша ветка содержала последние изменения из основной ветки и не было конфликтов при создании MR.*
