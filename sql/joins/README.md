# SQL Data Joining

## Overview

В этом разделе представлены SQL-запросы, демонстрирующие работу с объединением данных из нескольких таблиц.

Объединение таблиц является одной из ключевых задач аналитика данных, так как данные в реальных базах обычно хранятся в нормализованном виде и распределены между несколькими таблицами.

Для выполнения аналитических задач необходимо уметь правильно соединять таблицы, извлекая из них связанную информацию.

Все задания выполнены на основе учебного датасета **Karpov.Courses**.

# Key SQL Concepts

## JOIN

`JOIN` используется для объединения строк из двух или более таблиц на основе общего ключа.

Основные типы соединений:

### INNER JOIN

Возвращает только те строки, для которых найдено совпадение в обеих таблицах.

Используется, когда необходимо работать только с пересечением данных.

---

### LEFT JOIN

Возвращает все строки из **левой таблицы** и совпадающие строки из правой таблицы.

Если совпадений нет — значения из правой таблицы будут `NULL`.

Часто используется для:

* поиска пользователей без заказов
* поиска товаров без продаж
* анализа неполных данных

---

### RIGHT JOIN

Возвращает все строки из **правой таблицы** и совпадения из левой таблицы.

Используется реже, но может быть полезен при анализе справочников или каталогов.

---

### FULL OUTER JOIN

Возвращает все строки из обеих таблиц.

Используется, когда необходимо увидеть:

* совпадающие записи
* записи без совпадений в обеих таблицах

---

## JOIN with Multiple Tables

SQL позволяет объединять сразу несколько таблиц.

Это особенно важно в аналитике, когда данные распределены между:

* таблицей клиентов
* таблицей заказов
* таблицей товаров
* таблицей действий пользователей

Такие запросы позволяют строить полноценные аналитические выборки.

---

## UNION

`UNION` используется для объединения результатов нескольких запросов.

Особенности:

* объединяемые запросы должны иметь одинаковое количество столбцов
* типы данных должны совпадать

### UNION

Удаляет дубликаты строк.

### UNION ALL

Сохраняет все строки, включая дубликаты.

`UNION ALL` работает быстрее и чаще используется в аналитических задачах.
