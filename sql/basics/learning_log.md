# **1 урок: Какие навыки приобретем?**
- быстро и эффективно работать с таблицами 
- писать sql-запросы с фильтрацией и группировкой
- объединять таблицы с помощью join
- составлять сложные запросы с подзапросами и оконными функциями

## С какими данными работает аналитик?
- ЧТО: *Данные* - это зафиксированные факты о событиях, действиях, объектах.
- ГДЕ: бизнес, наука, медицина, обраховани и тд
- ЗАЧЕМ: 1) понять поведение на текущий момент 2) улучшать процессы 3) принимать решения на основе фактов

## С какими интсрументами работает аналитик?
1. Электронные таблицы
>Небольшие объемы данных
2. SQL (Structured Query Language )
>Язык запросов к базам данных. 
>Позволяет выбирать, фильтровать, объединять и агрегировать данные.
>Основа работы аналитика с хранилищами данных

3. Python
> Универсальный язык для анализа и автоматизации.
Позволяет работать с большими данными, строить модели, визуализировать.
Используется для продвинутой аналитики и машинного обучения
4. ВI-системы (Power BI, Tableau, Looker и др.) 
> Инструменты для создания интерактивных дашбордов и визуализаций.
Помогают бизнесу отслеживать ключевые метрики в реальном времени.
Упрощают принятие решений на основе данных
## Зачем аналитику SQL?
- Доступ к "сырым" данным без искажений и агрегаций;
- Гибкость при анализе: можно строить любые выборки и срезы;
- Возможность обрабатывать  большие объёмы данных;
- Основа для продвинутой аналитики – моделей, дашбордов, А/В-тестов.

# **2 урок: Основы SQL: операторы, типы данных и базовые функции**
Будем учиться на основе того, как будто мы уже работаем в вымышленном маркетплейсе. Данные в данном случае хранятся в определленых таблицах:
1. Customers (клиенты) - вся информация про клиентов(город, айди,пол, день рождение и тд)
2. Customer_actions (действия конкретного клиента) - зашел на сайт, открыл карточки, добавил в корзину и тд - обычно называют *"логами"* таблицы, зависящие от времени
3. Orders (заказы) - номер заказа, время, когда создался заказ, когда доставился, примерное ожидаемое время доставки и статус заказа
4. Order_items(информация по заказу) - id заказа, что было выбрано в заказе, их стоимость и тд
5. Products(продукты) - конкретная информация по продукту - вес, размеры, стоимость, бренд и тд
> Вся суть в том, что нет единой таблицы и нужно уметь обращаться к разным таблицам, чтобы найти нужную информацию


## Key Concepts
- SELECT -> FROM-> ORDER BY -> LIMIT
- Основные типы данных: 
    1. Числовые: INTEGER(целые), NUMERIC(числа с точкой)
    2. Символьные: VARCHAR("строка в кавычках")
    3. Типы данных даты и времени: DATE, TIME, TIMESTAMP(дата+время), INTERVAL(промежуток времени)
    4.Логический тип: BOOLEAN(True или False)
    5. Массивы (Arrays): позволяют хранить несколько значений одного и того же типа в одной ячейке таблицы
- Если деление целых чисел, получим целое число.
## Important Commands
- ORDER BY() (ASC - по возрастанию,DESC - по уменьшению) - сортировка дынных
- ROUND() - округление до n-го знака
- POWER(стобец, степень) - возведение в степень
- SQRT() - корень 2 степени 
- ABS() - модуль числа
- CEIL() - округление числа вверх. Пример: CEIL(3.1) = 4
- FLOOR() - округление числа вниз. Пример: FLOOR(3.9) = 3
- CONCAT(str1, str2, ...) - помогает объединить эти столбцы и получить столбец
- UPPER(str) и LOWER(str) - используются, чтобы привести данные к единому регистру во избежание дублей
- LENGTH(str) - Функция используется для оценки количества символов в строке.
- SUBSTRING(str, start, length) -  Эта функция имеет 3 аргумента: str — сама строка, start — с какого по счёту символа начинать извлечение, length — длина извлекаемой строки
- REPLACE(str, old_substr, new_substr) - Эта функция имеет 3 аргумента: str — сама строка, old_substr — что заменяем, new_substr — на что заменяем.
- TRIM(str) - Функция используется для удаления пробелов.
- SPLIT_PART(str, разделитель, позиция) - Имеет 3 аргумента: str — сама строка, разделитель — по чему идёт разделение, позиция — какой по счёту фрагмент вернуть. Пример: SELECT SPLIT_PART('user@domain.com', '@', 2) = 'domain.com'
