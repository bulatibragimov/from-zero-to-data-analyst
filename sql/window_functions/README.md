# SQL Window Functions

## Overview

В этом разделе рассматривается использование **оконных функций (Window Functions)** в SQL для аналитики данных.

Оконные функции позволяют выполнять вычисления по набору строк, связанных с текущей строкой, **не агрегируя данные**, как это делает `GROUP BY`. Благодаря этому можно сохранять исходные строки таблицы и одновременно вычислять дополнительные аналитические метрики.

Все задания основаны на тренировочном датасете **Karpov.Courses SQL training program**.

Цель данного раздела — продемонстрировать умение:

* использовать оконные функции для аналитических вычислений
* выполнять ранжирование и нумерацию строк
* анализировать последовательности данных
* работать со скользящими метриками

---

# Key SQL Concepts

## Window Functions

Оконные функции позволяют вычислять агрегаты **для каждой строки**, при этом сохраняя другие столбцы результата.

В отличие от `GROUP BY`, который агрегирует строки, оконные функции позволяют добавлять аналитические метрики без потери исходных данных.

---

## PARTITION BY

`PARTITION BY` используется для разделения данных на группы (партиции), внутри которых применяется оконная функция.

Это похоже на `GROUP BY`, но строки **не агрегируются**, а остаются в результирующей таблице.

Например:

* анализ заказов по клиентам
* ранжирование товаров внутри категории
* расчёт метрик по группам пользователей

---

## ORDER BY in Window Functions

`ORDER BY` внутри оконной функции задаёт порядок строк **внутри каждой партиции**.

Это особенно важно для функций:

* `ROW_NUMBER`
* `RANK`
* `DENSE_RANK`
* `LAG`
* `LEAD`

Порядок строк определяет, как будет выполняться нумерация или сравнение значений.

---

## Ranking Functions

Функции ранжирования используются для нумерации строк внутри группы.

Наиболее часто используются:

* `ROW_NUMBER()` — присваивает уникальный номер каждой строке
* `RANK()` — одинаковые значения получают одинаковый ранг
* `DENSE_RANK()` — ранги без пропусков

Эти функции широко применяются в аналитике для:

* поиска топ-N значений
* определения позиции в рейтинге
* анализа лидеров продаж

---

## LAG and LEAD

Функции `LAG()` и `LEAD()` позволяют получать значения **из предыдущей или следующей строки**.

Это особенно полезно при анализе:

* временных рядов
* последовательности заказов
* изменений метрик во времени

Например:

* сравнение текущего значения с предыдущим
* вычисление изменения показателей между периодами

---

## Window Frames (ROWS)

Ключевое слово `ROWS` позволяет определить **границы окна**, внутри которого выполняется вычисление.

Например:

* текущая строка
* несколько предыдущих строк
* диапазон строк вокруг текущей

Это используется для вычисления:

* скользящих сумм
* скользящих средних
* накопительных метрик

---

## Moving Averages

Скользящее среднее используется для сглаживания данных и выявления основных тенденций.

Такая техника применяется в аналитике для:

* анализа динамики продаж
* изучения поведения пользователей
* анализа временных рядов

Скользящие метрики помогают уменьшить влияние случайных колебаний и лучше понять общую тенденцию данных.

---

Оконные функции являются одним из самых мощных инструментов SQL-аналитики и позволяют решать сложные аналитические задачи с минимальным количеством кода.
