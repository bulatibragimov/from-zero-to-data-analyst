## Covered SQL Concepts

### 1. WHERE — фильтрация данных

Конструкция `WHERE` используется для отбора строк по заданным условиям.

Применялись:

* Числовые фильтры
* Строковые фильтры
* Фильтрация по датам
* Комбинация условий через `AND` / `OR`

Пример:

```sql
WHERE order_status != 'Returned'
AND order_delivered_customer_time IS NOT NULL
```

---

### 2. BETWEEN и IN

**BETWEEN** — фильтрация по диапазону значений
**IN** — фильтрация по списку значений

Примеры:

```sql
WHERE order_created_time BETWEEN '2024-01-15' AND '2024-03-03'

WHERE product_category_name IN ('Электроника','Одежда','Сад')
```

---

### 3. LIKE — частичные совпадения

`LIKE` используется для поиска по шаблону.

* `%` — любое количество символов
* `_` — один любой символ

Пример:

```sql
WHERE LOWER(product_brand) LIKE '%фото%'
```

---

### 4. Работа с NULL

Корректная обработка отсутствующих значений:

* `IS NULL`
* `IS NOT NULL`
* `COALESCE()`

Пример:

```sql
COALESCE(customer_city, CAST(customer_zip_code AS VARCHAR))
```

Использование `COALESCE` позволяет заменить `NULL` значением по умолчанию для корректного вывода в отчетах.

---

### 5. Работа с датами

Использованные функции:

* `DATE_PART()` — извлечение года, месяца, дня
* `CAST()` — преобразование типов данных
* Работа с `INTERVAL`
* Вычисление разницы дат

Пример:

```sql
DATE_PART('year', created_at)
```

---

### 6. CASE — условная логика

Использование `CASE` для бизнес-классификации данных.

Примеры:

* Определение региона клиента
* Классификация брендов
* Определение статуса доставки (вовремя / опоздал)
