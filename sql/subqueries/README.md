# SQL Subqueries

## Overview

В этом разделе рассматривается использование **подзапросов (Subqueries)** в SQL для решения аналитических задач.

Подзапросы позволяют выполнять вложенные вычисления и использовать результаты одного запроса внутри другого. Это один из ключевых инструментов при работе с аналитическими запросами и сложной фильтрацией данных.

Все задания основаны на тренировочном датасете **Karpov.Courses SQL training program**.

Цель данного раздела — продемонстрировать умение:

* использовать вложенные запросы для фильтрации данных
* применять подзапросы для вычисления сравнительных метрик
* комбинировать подзапросы с агрегатными функциями
* использовать CTE для улучшения структуры SQL-запросов

---

# Key SQL Concepts

## Subqueries in WHERE

Подзапросы в блоке `WHERE` используются для более продвинутой фильтрации данных.

Они позволяют сравнивать значения строк с результатами других запросов, например:

* фильтрация по максимальному значению
* сравнение со средними показателями
* выбор строк на основе связанных таблиц

---

## Subqueries in FROM / JOIN

Подзапрос может использоваться как временная таблица внутри блока `FROM`.

Это позволяет:

* выполнять предварительную агрегацию данных
* создавать промежуточные таблицы для анализа
* упрощать структуру сложных аналитических запросов

---

## Subqueries in SELECT

Подзапросы могут использоваться внутри блока `SELECT` для добавления дополнительных метрик в результат запроса.

Например:

* сравнение метрик категории со средним значением по всей таблице
* добавление глобальных статистик к каждой строке результата

В аналитических задачах такие вычисления часто выполняются также с помощью **оконных функций (Window Functions)**.

---

## EXISTS and IN

Операторы `EXISTS` и `IN` используются для проверки наличия связанных данных.

`IN` проверяет, содержится ли значение в списке, возвращаемом подзапросом.

`EXISTS` проверяет, существует ли хотя бы одна строка, удовлетворяющая условию.

---

## Common Table Expressions (CTE)

CTE (Common Table Expression) — это именованный подзапрос, объявляемый с помощью конструкции `WITH`.

Преимущества CTE:

* улучшает читаемость SQL-кода
* позволяет разбивать сложные запросы на логические блоки
* позволяет переиспользовать промежуточные результаты внутри запроса

CTE широко используется в аналитике данных и BI-инструментах.
