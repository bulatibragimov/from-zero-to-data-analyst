# Key SQL Concepts

## Aggregation Functions

Агрегатные функции позволяют сворачивать множество строк в одно значение и используются для расчёта аналитических метрик.

Основные функции:

* **COUNT()** — подсчёт количества строк
* **SUM()** — сумма значений
* **AVG()** — среднее значение
* **MIN()** — минимальное значение
* **MAX()** — максимальное значение

Такие функции широко используются в аналитике для расчёта:

* количества пользователей
* среднего чека
* количества заказов
* диапазона цен на продукты

---

## GROUP BY

Конструкция **GROUP BY** используется для группировки строк по определённому признаку.

Это позволяет рассчитывать агрегированные показатели для различных категорий данных.

Типичные задачи:

* количество клиентов по городам
* количество заказов по пользователям
* средний вес товаров по брендам
* количество событий по типу действия пользователя

---

## WHERE vs HAVING

Обе конструкции используются для фильтрации данных, но применяются на разных этапах выполнения запроса.

**WHERE**

Фильтрует строки **до выполнения агрегации**.

Пример использования:

* выбрать заказы за определённый период
* выбрать только доставленные заказы
* исключить строки с NULL значениями

**HAVING**

Фильтрует данные **после выполнения группировки и агрегации**.

Используется для условий над агрегированными значениями.

Пример:

* оставить только бренды со средним весом товара меньше 200 г
* оставить пользователей с более чем 5 заказами

---

## Conditional Aggregation

Иногда необходимо посчитать разные категории данных в одном запросе.

Для этого используются конструкции:

* **CASE WHEN**
* **FILTER**

Пример аналитической задачи:

* посчитать количество доставленных заказов
* посчитать количество возвратов
* сравнить разные статусы заказов
