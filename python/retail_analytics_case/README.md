# Python Pandas Analytical Tasks — Retail Transaction Analysis

## Overview

Данный проект представляет собой комплексный анализ транзакционных данных розничного магазина. В рамках работы решены продуктовые задачи: от первичной обработки данных до выявления аномалий и сегментации активных пользователей.

**Цель проекта** — продемонстрировать навыки аналитика данных на реальных бизнес-задачах: очистка данных, фильтрация, агрегация, расчёт метрик, построение сводных таблиц и визуализация.

Проект выполнен в рамках курса аналитики данных и содержит **два независимых датасета**:
1. Транзакции интернет-магазина (заказы, товары, клиенты, страны)
2. Логи транзакций пользователей с временными метками

## Business Tasks Solved

### 1. Data Cleaning & Preparation
- Загрузка данных из ZIP-архива с кодировкой `ISO-8859-1`
- Удаление дубликатов (обнаружено и удалено **5 228 дубликатов**)
- Фильтрация отменённых транзакций по префиксу `C` в `InvoiceNo`
- Приведение типов данных, работа с пропусками

### 2. Customer Segmentation (Germany Active Users)
- Отбор клиентов из Германии
- Расчёт количества уникальных заказов на пользователя (`nunique`)
- Определение порога активности = **80-й процентиль**
- Выявлены **топ-активные пользователи Германии** для дальнейшего анализа

### 3. Product Popularity Analysis
- Анализ частоты добавления товаров в корзину (без учёта количества)
- Исключение услуги доставки (`POST`)
- Определён **самый популярный товар** среди активных пользователей Германии

### 4. Revenue Analysis — Top Orders
- Создание колонки `Revenue = Quantity * UnitPrice`
- Агрегация выручки по заказам (`groupby + sum`)
- **Выявлены топ-5 заказов по выручке** (номера заказов получены)

### 5. Transaction Status Analysis
- Подсчёт транзакций по статусам (`successfull`, `error`, `cancelled`)
- Визуализация барплотом (seaborn)
- Расчёт статистики успешных транзакций на пользователя:
  - Среднее = **~66 транзакций**
  - Максимум = **112 транзакций**
  - 25% пользователей совершили менее 61 транзакции
- Построение гистограммы распределения

### 6. Pivot Table & Anomaly Detection
- Создание сводной таблицы: строки = минуты, столбцы = пользователи, значения = число операций
- Заполнение пропусков нулями
- **Обнаружена ошибка в данных**: вместо минут с начала дня была извлечена только минутная часть
- Исправление: создана колонка `true_minute = час * 60 + минута`
- Построена корректная гистограмма распределения транзакций по времени

## Key Pandas Concepts Demonstrated

| Метод / Приём | Где использован |
|---------------|----------------|
| `pd.read_csv(compression='zip', encoding='ISO-8859-1')` | Загрузка данных из ZIP |
| `drop_duplicates()` | Удаление дубликатов |
| `str.startswith()` + `~` | Фильтрация отменённых заказов |
| `query()` | Читаемая фильтрация по условиям |
| `groupby() + agg({'col': 'nunique'})` | Количество уникальных заказов на клиента |
| `quantile(q=0.8)` | Порог активности пользователей |
| `isin()` | Отбор по списку ID |
| `value_counts()` | Частота товаров в корзине |
| `assign()` | Создание колонки Revenue |
| `groupby() + agg({'Revenue': 'sum'})` | Выручка по заказам |
| `sort_values(ascending=False).head()` | Топ-5 заказов |
| `groupby().size()` | Подсчёт операций по группам |
| `pivot(index, columns, values).fillna(0)` | Сводная таблица с заполнением пропусков |
| `dt.hour`, `dt.minute` | Извлечение временных компонентов |
| `sns.barplot()`, `sns.displot()` | Визуализация распределений |
