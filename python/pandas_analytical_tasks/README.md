# Python Pandas Analytical Tasks

## Overview
В данном разделе представлены решения аналитических задач с использованием библиотеки pandas.

Проект охватывает полный базовый цикл работы аналитика с данными: от загрузки и первичной обработки данных до их анализа, построения метрик и извлечения бизнес-инсайтов.

Задачи выполнены на тренировочных датасетах в рамках курса аналитики данных.

Цель проекта — продемонстрировать практические навыки анализа данных и подготовки аналитических выводов.

---

## Project Structure

- `analytical_tasks.ipynb` — базовые операции с данными (загрузка, фильтрация, агрегации)
- `more_pandas_tasks.ipynb` — расширенная работа с pandas (группировки, метрики, сегментация)
- `data_exploration_and_metrics_creation.ipynb` — исследование данных (EDA) и построение аналитических метрик

---

## Business Tasks

В рамках проекта были решены следующие аналитические задачи:

### Data Exploration (EDA)
- изучение структуры датасета
- анализ типов данных и пропусков
- исследование распределения ключевых метрик
- выявление аномалий и особенностей данных

---

### Data Cleaning & Preparation
- приведение типов данных (в том числе работа с датами)
- переименование колонок для удобства анализа
- обработка некорректных значений

---

### Revenue & Metrics Analysis
- расчёт ключевых метрик (выручка, средние значения)
- анализ распределения выручки
- сравнение показателей между категориями

---

### Customer & Product Analysis
- анализ популярности продуктов (`value_counts`)
- сегментация данных по категориям
- выявление наиболее прибыльных направлений

---

### Filtering & Business Logic
- фильтрация данных по статусам (например, успешные операции)
- исключение нерелевантных данных (отмены, нулевые значения)
- применение условий для анализа

---

### Grouping & Aggregation
- расчёт метрик по группам (`groupby`)
- анализ данных по категориям (продукты, пользователи и т.д.)
- сравнение агрегированных показателей

---

### Feature Engineering
- создание новых колонок (например, преобразование выручки)
- подготовка данных для анализа
- упрощение дальнейших расчётов

---

### Analytical Pipelines
- построение цепочек обработки данных (method chaining)
- объединение фильтрации, агрегации и сортировки в единый процесс

---

## Key Pandas Concepts

В проекте активно использовались:

- `read_csv()` — загрузка данных
- `query()` — фильтрация
- `groupby()` — группировка
- `agg()` — агрегации
- `value_counts()` — анализ категорий
- `sort_values()` — сортировка
- работа с датами (`parse_dates`)
- векторизация операций
- method chaining

---

## Key Insights

- корректная фильтрация данных напрямую влияет на итоговые метрики
- агрегированные показатели могут существенно отличаться от средних значений по всей выборке
- распределение данных помогает выявлять ключевые источники выручки
- использование группировок позволяет выявлять сильные и слабые категории
- метод chaining упрощает построение аналитических процессов и делает код более читаемым
